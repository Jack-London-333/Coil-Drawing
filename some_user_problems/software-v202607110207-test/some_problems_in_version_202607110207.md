## 问题五

每次任务需要创建一个目录（也可以进入一个已有的目录），这个目录就是当前软件的工作空间。目录里面包含一个 .txt 的各项参数的配置文件，用户既可以在软件内部修改，同时同步到外部的配置文档，用户也可以在这个 .txt 中进行修改。此txt文件的名称由你定夺，如config.txt。

当然用户也可以创建一个目录，然后选择一个 .txt 配置文件（如ABC.txt，此名字并不符合软件的“config.txt”这个标准名称），软件会把这个文件复制到当前目录 并改名为config.txt。

软件也要有一个默认的 config.txt 作为模板，放在一个地方。

## 问题六

参数的数值精度问题。定子铁芯内径D2、铁芯轴向长度LC...这些数值精度只能达到 0.1 mm，我认为 除了那些只能是整数的参数 其他数值精度参数都要达到10^(-8) mm的精度，你觉得呢？应该多大的精度合理？

## 问题七

软件UI界面的数值窗口上，鼠标滚轮滑动会带动数值变化，这个“滚轮带动数值变化”的功能太鸡肋了，很容易误触，需要去掉。
